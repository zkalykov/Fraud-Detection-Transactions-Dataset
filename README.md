# 💳 Fraud Detection Transactions Analysis

This project explores and analyzes synthetic financial transaction datasets to understand patterns related to fraudulent activities.

## 📂 Datasets Used

- **Fraud Detection Transactions Dataset**  
  Synthetic dataset containing labeled fraud and non-fraud transactions.

- **PaySim Dataset**  
  Simulated mobile money transactions designed for fraud detection research.

## ⚙️ Project Workflow

1. **Data Acquisition**
   - Datasets are downloaded using `kagglehub`

2. **Data Loading**
   - CSV files are loaded into Pandas DataFrames

3. **Exploratory Data Analysis (EDA)**
   - Preview of datasets (`head`, structure)
   - Dataset size (rows, columns, memory usage)
   - Feature inspection

4. **Dataset Comparison**
   - Structural and statistical comparison between datasets
   - Evaluation of dataset scale and usability

5. **Feature Selection (Optional)**
   - Removal of unnecessary columns
   - Focus on features relevant to fraud detection

## 📊 Key Insights

- Large-scale transaction datasets require memory-efficient handling  
- Location, transaction type, and amount are important indicators of fraud  
- Synthetic datasets are useful for experimentation but may lack real-world complexity  

## 🛠️ Technologies Used

- Python  
- Pandas  
- KaggleHub  
- Jupyter Notebook / Google Colab  

## 🚀 Goal

To build a foundational understanding of fraud detection datasets and prepare data for future machine learning models.
